Git使用

Troy Zhang

Git简介

  • 分布(/集中)式版本管理系统
  • Linux
  • branch
  • tag
  • GitHub, fork, pull request, issue, CI, comments

特点

  • 不需要联机
  • 每台机器都是一个版本库，可提交到统一的服务器
  • 本地包含完整的commit日志
  • 可创建本地分支，在分支上开发

安装

  • Mac OS X
  • Windows

配置

# 查看配置列表
$ git config -l

# 配置git全局邮箱
$ git config --global user.email "xxxx@163.com"

# 配置git全局用户名
$ git config --global user.name "用户名"

常用命令1-本地管理

# 初始化项目--全新的项目
$ git init

# 初始化项目--已存在的项目
$ git clone ssh://xdzhangm@iss110301000305/LBTravel.git

# 添加新文件
$ git add .

# 提交代码到本地版本库
$ git commit -am "xxxx"

# 查看日志
$ git log

# 获取某个时间点的代码
$ git checkout 3286ea4

# 查看当前branch
$ git branch

.gitignore

# 哪些文件是不需要版本管理的？
$ cat .gitignore

.svn
.svn/*
*svn*
.DS_Store
wc.db
build
build/*

常用命令2-branch管理

# 从当前分支创建新的分支
$ git checkout -b dev

# 提交代码
$ git commit -am "....."

# 切换分支
$ git checkout master

# 合并代码
$ git merge dev

tags

# 创建tag
$ git tag -a v1.0.0 -m "......."

# 查看tag列表
$ git tag -l

# 提交tag到远程服务器
$ git push origin --tags

# 删除本地tag
$ git tag -d v1.0.0

# 删除远程tag
$ git push origin :refs/tags/v1.0.0

SSH配置

# 创建公钥、私钥
$ ssh-keygen -t rsa -C "xxx@isstech.com"

# 多ssh管理（一般不需要）
$ vim ~/.ssh/config
Host github
    HostName github.com
    User github_user
    IdentityFile ~/.ssh/id_rsa_github
$ ssh-add -l

# 上传公钥（在github/gitlab服务器上配置，用户从本地推送代码的时候服务器会验证用户的身份）
$ cat ~/.ssh/id_rsa.pub

常用命令3-推送代码到服务器

# 添加远程库
$ git remote add origin ssh://xdzhangm@iss110301000305/YeSanPo.git

# 查看远程库
$ git remote -v

# 提交到远程库
$ git push origin master

常用命令4-从服务器更新代码

# 从远程服务器获取最新代码
$ git fetch origin master

# 从远程服务器获取最新代码，并且合并代码
$ git pull origin master
